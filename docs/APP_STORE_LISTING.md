## Data safety & declarations (checklist for Play “Data safety”)

When you fill out Google’s Data safety form, align answers with your real backend and this privacy policy. Typical categories for an app like Second Chat include:

| Area | Often declared |
|------|----------------|
| **Personal info** | Name, email (e.g. from Google account), User IDs |
| **Financial info** | Usually “No” unless you sell in-app purchases through Play Billing in this app |
| **Location** | Only if you collect precise or approximate location beyond IP—adjust to your truth |
| **Photos / videos** | If you allow uploads; otherwise “No” |
| **App activity** | In-app actions, interactions |
| **Web browsing** | Only if in-app browser sends URLs to you |
| **Device or other IDs** | Device ID, push token, advertising ID only if you use ads |

**Data shared** — Firebase/Google; your API; OAuth providers as applicable.  
**Security practices** — Encryption in transit; optional encryption at rest for certain fields on server.  
**Deletion** — Describe how users request account or data deletion.
